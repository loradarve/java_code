
# 《计算机和通信网络》Ch14 隧道、VPN与MPLS 习题答案（紧凑打印版）

## 一、判断题（第1～4题）

**1. Tunneling是一种新的网络层协议。** **False**  
*解析*：隧道（Tunneling）是一种封装技术，而非新协议；它将一个协议的数据包封装在另一个协议中传输，利用现有网络层协议（如IP）实现。

**2. MPLS转发基于IP地址查表。** **False**  
*解析*：MPLS基于**标签**转发，在边缘路由器进行IP到标签映射，核心路由器只根据标签交换，不查看IP地址。

**3. 流量工程可以忽略链路带宽约束。** **False**  
*解析*：流量工程（Traffic Engineering, TE）的核心正是考虑链路带宽、时延等约束，通过显式路径优化网络资源利用率。

**4. MPLS VPN依靠IPsec加密实现隔离，与传统VPN安全机制相同。** **False**  
*解析*：MPLS VPN主要通过VRF（路由表隔离）和标签转发实现逻辑隔离，不依赖IPsec加密；传统VPN（如IPsec VPN）依赖加密/认证实现安全。

## 二、单选题（第5～8题）

**5. Tunneling的核心机制是：**  
A. 加密 B. **封装** C. 路由 D. 调度  
**答案：** **B**  
*解析*：隧道的本质是将原始数据包封装在新的报头中，在隧道两端进行解封装，从而实现在不同网络间透明传输。

**6. MPLS标签中，用于标识QoS优先级的字段是：**  
A. Label B. **EXP/TC** C. S D. TTL  
**答案：** **B**  
*解析*：MPLS标签中有3位EXP（Experimental）字段，后更名为TC（Traffic Class），用于标识服务类别和优先级。

**7. MPLS网络中，负责弹出标签、恢复原始IP包的节点是：**  
A. Ingress LSR B. Transit LSR C. **Egress LSR** D. 传统IP路由器  
**答案：** **C**  
*解析*：出口LSR（Egress LSR）在将分组转发给下游非MPLS网络前，弹出顶层标签，恢复原始IP分组。

**8. VRF的作用是：**  
A. 提升带宽 B. **实现隔离** C. 加密数据 D. 提高速率  
**答案：** **B**  
*解析*：VRF（Virtual Routing and Forwarding）是MPLS VPN中为每个VPN实例维护独立路由表，实现不同客户间路由隔离。

## 三、填空题（第9～13题）

**9. Tunneling的核心思想是** **[封装（Encapsulation）]**。  
*解析*：将原始分组作为负载，加上新的外层协议头，在隧道中传输。

**10. VPN本质是建立在** **[公共（或共享）]** **网络上的逻辑网络**。  
*解析*：VPN利用隧道技术在公共互联网或运营商网络上构建私有、隔离的逻辑网络。

**11. MPLS的全称是** **[多协议标签交换（Multi-Protocol Label Switching）]**。  
*解析*：MPLS是一种结合二层交换和三层路由的技术，使用短标签加速转发。

**12. MPLS标签栈中，标识栈底的字段是** **[S（Bottom of Stack）]**。  
*解析*：标签栈中每个标签的S位（1位），为1表示该标签为栈底，之后为IP载荷。

**13. 流量工程核心优化对象是** **[网络资源利用率]**（或“链路带宽利用率”）。  
*解析*：流量工程通过调整路径来避免拥塞，均衡负载，提高带宽利用率。

## 四、多选题（第14～17题）

**14. Tunneling作用包括：** A.协议兼容 B.地址隔离 C.透明通信 D.加密  
**答案：** **A**、**B**、**C**、**D**（加密是可选功能，但隧道可配合IPsec实现）  
*解析*：隧道可封装非IP协议（协议兼容），隐藏内部地址（地址隔离），提供端到端透明传输，并可结合加密增强安全。

**15. MPLS标签操作包含：** A.Push B.Swap C.Pop D.Delete  
**答案：** **A**、**B**、**C**  
*解析*：压入（Push）、交换（Swap）、弹出（Pop）是基本标签操作；Delete不是标准术语。

**16. TE考虑因素包括：** A.带宽 B.时延 C.路由协议 D.负载  
**答案：** **A**、**B**、**C**、**D**  
*解析*：流量工程需考虑链路带宽、时延约束，调整路由协议参数，并监控负载分布。

**17. 相比传统VPN，MPLS VPN的优势有：** A.扩展性强 B.管理简单 C.支持大规模多租户 D.依赖公网加密  
**答案：** **A**、**B**、**C**  
*解析*：MPLS VPN基于路由隔离，无需隧道加密，扩展性好，支持成千上万的租户；D错误，不依赖公网加密。

## 五、简答题（第18～21题）

**18. 什么是Tunneling？其工程意义是什么？**  
**答**：Tunneling是将一个数据包（如IPv6、IPX）封装在另一个协议（如IPv4）中传输的技术。工程意义：实现异构网络互通（如IPv4 over IPv6）、穿越防火墙、构建VPN、提供安全性（结合IPsec）。

**19. VPN的两种典型类型及应用场景**  
**答**：  
- **站点到站点VPN**：连接两个局域网（如企业总部与分支），使用IPsec或MPLS VPN。  
- **远程接入VPN**：单个用户接入企业内网，使用SSL VPN或IPsec。应用场景包括远程办公、多云互联。

**20. MPLS为什么比IP转发更高效？**  
**答**：IP转发需每跳最长前缀匹配路由表（复杂且慢）；MPLS在入口打标签，核心路由器仅根据标签快速交换（类似二层硬件交换），减少查表开销，支持流量工程和快速重路由。

**21. 简述MPLS标签栈（Label Stack）的作用。**  
**答**：标签栈支持多层标签嵌套，顶层标签用于当前路径转发，下层标签用于后续处理（如VPN、隧道）。作用：实现层次化转发（如MPLS VPN中，上层标签指示下一跳，下层标签标识VPN成员）。

## 六、论述题（第22～25题）

**22. 论述Tunneling → VPN → MPLS的技术演进路径**  
**答**：  
- **Tunneling**：封装技术基础（如GRE）。  
- **VPN**：利用隧道构建私有网络（如IPsec VPN、SSL VPN），但面临扩展性、管理复杂问题。  
- **MPLS**：提供标签交换，原生支持VPN（BGP/MPLS IP VPN），具备良好扩展性和流量工程能力，成为运营商级VPN主流技术。  
**演进核心**：从简单封装到安全隔离，再到高效、可扩展的多租户方案。

**23. 论述VPN安全机制（认证、加密、完整性）的作用与协同原理**  
**答**：  
- **认证**：验证通信双方身份（如IKE中的证书或预共享密钥）。  
- **加密**：保护数据机密性，防止窃听（如AES）。  
- **完整性**：防止数据篡改（如SHA+HMAC）。  
**协同**：认证建立信任，加密保护数据，完整性校验确保未被修改。三者结合实现VPN的CIA（机密性、完整性、可用性）安全目标。

**24. 论述MPLS面向连接的本质**  
**答**：MPLS通过LDP或RSVP-TE建立LSP（标签交换路径）。LSP类似虚电路，在入口为分组分配标签，沿途LSR按标签转发，无需逐跳查询路由。这种面向连接的特性支持流量工程、快速重路由和QoS保证。

**25. 论述MPLS对现代运营商网络的核心价值**  
**答**：  
- **高效转发**：标签交换比IP路由快。  
- **流量工程**：避免拥塞，优化链路利用率。  
- **VPN服务**：提供扩展性极强的L3/L2 VPN。  
- **QoS**：支持EXP位区分服务等级。  
- **可靠性**：快速重路由（FRR）实现50ms保护切换。  
MPLS是运营商网络的核心技术。

## 七、分析题（第26～29题）

**26. 分析为什么IPv6需要Tunneling**  
**答**：IPv6部署过渡期内，大量网络仍为IPv4。隧道技术将IPv6分组封装在IPv4中传输，连接IPv6孤岛，无需升级所有设备。常用隧道协议包括6to4、Teredo、GRE over IPv4。

**27. 分析MPLS为何适合运营商网络**  
**答**：  
- 边缘复杂，核心简单：边缘做IP路由和标签映射，核心仅标签交换。  
- 支持多业务：L2VPN、L3VPN、互联网接入同网承载。  
- 可扩展：VRF隔离客户，标签栈实现层次化。  
- 可靠性高：LDP和IGP联动，故障快速收敛。

**28. 分析Label Switching与IP Routing差异**  
**答**：  
| 特性 | Label Switching | IP Routing |
|------|----------------|-------------|
| 查表 | 标签（短定长） | IP地址（变长，最长前缀匹配） |
| 转发 | 快速硬件交换 | 软件路由查表 |
| 路径 | 建立LSP（面向连接） | 逐跳独立决策 |
| 应用 | TE、VPN、快速重路由 | 尽力而为互联 |

**29. 分析MPLS流量工程为何要采用“显式路径”而非IGP最短路径**  
**答**：IGP最短路径可能导致部分链路拥塞，其他链路闲置。显式路径（如RSVP-TE创建的LSP）可人为控制路径，避开拥塞链路，均衡负载，满足带宽约束，实现全局优化。

## 八、辨析题（第30～33题）

**30. VPN是否等同于加密通信？**  
**答**：不等同。VPN的核心是逻辑隔离和私有性，加密是其中一种安全手段。例如MPLS VPN不加密，仅通过路由隔离实现隐私；IPsec VPN则依靠加密。因此VPN ≠ 加密。

**31. MPLS是否替代IP？**  
**答**：不替代。MPLS叠加在IP之上，仍依赖IP路由协议（OSPF、IS-IS）分发路由信息，MPLS用于优化转发和业务承载，而非取代IP。

**32. 辨析：“MPLS标签与IP地址的作用完全相同”。**  
**答**：错误。IP地址标识节点和网络位置，用于路由；MPLS标签标识LSP路径，只在本地有效，且可堆叠。标签简化转发，但不替代寻址。

**33. 辨析：“隧道技术只能用于安全加密场景”。**  
**答**：错误。隧道初始用途是协议封装（如GRE），解决异构网络互通。加密是可选增强（如IPsec over GRE）。隧道还用于移动IP、IPv6过渡、VPN等多种非安全场景。

## 九、综合应用题（第34～37题）

**34. 设计一个企业跨城市网络方案，要求：安全 + 可扩展**  
**答**：采用MPLS VPN（运营商提供）或IPsec VPN。  
- **方案**：总部与分支间建立IPsec VPN隧道（用DMVPN），中心部署动态路由（BGP或OSPF），分支机构间通过隧道互联。可扩展：采用VPN路由反射器，支持新增站点无需改变中心配置。

**35. 多业务QoS网络设计**  
**答**：在MPLS网络中，使用DiffServ + MPLS EXP位。  
- 语音EF（低延迟），视频AF（保证带宽），尽力而为BE。  
- 边缘路由器标记EXP，核心基于EXP调度（如LLQ、WRED）。  
- 同时使用RSVP-TE预留带宽，确保语音质量。

**36. 描述数据包经Ingress→Transit→Egress的标签操作与转发流程**  
**答**：  
1. **Ingress LSR**：收到IP包，查FIB映射到FEC，压入（Push）标签，发送给下一跳。  
2. **Transit LSR**：收到带标签分组，查LFIB，交换（Swap）标签（替换为新标签），转发。  
3. **Egress LSR**：收到标签分组，查LFIB发现下一跳为非MPLS，弹出（Pop）标签，恢复IP包并路由。

**37. 运营商为100家企业提供隔离互联，设计MPLS VPN方案**  
**答**：采用BGP/MPLS IP VPN。  
- 每个企业分配唯一的RD（Route Distinguisher）和RT（Route Target），构建独立VRF。  
- 边缘PE设备维护各企业VRF路由表，使用MP-BGP传递路由。  
- 核心P设备仅做标签交换，不感知VPN。  
- 优点：支持成百上千租户，路由隔离，可扩展性强。
