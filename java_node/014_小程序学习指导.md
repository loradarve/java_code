### 📖 第6课：SCL-90 的因子分析

**目标**：理解如何从 90 个答案中挖掘结构化信息

**阅读**：[scl90.js](file:///c:/Users/48179/WeChatProjects/miniprogram-1/pages/scl90/scl90.js)

关键代码：
```javascript
const FACTORS = [
  { name: '躯体化', items: [0, 3, 11, 26, ...] },
  { name: '强迫症状', items: [2, 8, 9, 27, ...] },
  ...
]

// 计算每个因子分
const factorResults = FACTORS.map(f => {
    const sum = f.items.reduce((s, i) => s + answers[i], 0)
    const mean = sum / f.items.length  // 因子分 = 因子总分 ÷ 题目数
    return { name: f.name, mean, warning: mean > 3 }
})
```

> 🔑 理解：`items` 数组中的数字是 0-based 的题号索引，通过 `answers[i]` 取值，就是数据处理中经典的「索引映射」。

### 📖 第8课：分享机制深入

**目标**：理解跨用户的数据传递难题

**阅读**：[shareResult.js](file:///c:/Users/48179/WeChatProjects/miniprogram-1/pages/shareResult/shareResult.js)

核心矛盾：
```
A 设备的数据在 A 的 wx.Storage 中
B 打开分享链接时，读取的是 B 的 wx.Storage（空的）
→ 不能用 Storage 传数据，必须用 URL 参数
```

解决方案：
```javascript
// 分享者
onShareAppMessage() {
    return { path: `/pages/shareResult?type=phq9&score=8` }
}

// 接收者
onLoad(options) {
    const score = parseInt(options.score)  // 从 URL 中读取
    const treatment = buildPHQ9(score)      // 重新计算方案
}
```

> 🔑 理解：URL 参数只能传简单数据（数字、短字符串），方案文本在接收端通过 `buildTreatment(score)` 重新生成。
