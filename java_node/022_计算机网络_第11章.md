

# 《计算机和通信网络》Ch11 排队论与延迟分析 习题答案（紧凑打印版）

## 一、判断题（第1～7题）

**1. Little定理要求系统必须满足指数分布。** **False**  
*解析*：Little定理（L=λW）仅要求系统处于稳态且守恒，对到达过程、服务时间分布无任何限制，不要求指数分布。

**2. 排队系统的状态通常用系统中客户数表示。** **True**  
*解析*：排队系统的状态定义为系统中（包括正在服务和等待）的客户数量，这是描述系统状况的核心变量。

**3. M/M/1 中到达过程是泊松过程。** **True**  
*解析*：M/M/1的第一个“M”表示马尔可夫（Markov）到达过程，即泊松过程（到达间隔服从指数分布）。

**4. 当 ρ < 1 时系统一定没有排队。** **False**  
*解析*：ρ<1是系统稳定的必要条件，但即使ρ<1，由于随机性，仍然可能出现排队（队列非空），只是平均队列长度有限。

**5. M/M/1/b模型不会出现丢包。** **False**  
*解析*：M/M/1/b是有限缓冲模型（b为缓冲区容量），当系统客户数达到b时，新到达的客户将被丢弃（丢包）。

**6. Burke定理说明输出流与输入流统计特性相同。** **True**  
*解析*：Burke定理指出，对于M/M/1稳定系统，输出过程是泊松过程且与输入过程独立，速率相同。

**7. 自相似流量具有长程相关性。** **True**  
*解析*：自相似流量的关键特征之一是长程相关性（Long-Range Dependence），即自相关函数呈幂律衰减。

## 二、单选题（第1～7题）

**1. Little定理表达式为：**  
A. L=μW B. **L=λW** C. W=λ/L D. 无  
**答案：** **B**  
*解析*：Little定理：平均顾客数 L = 到达率 λ × 平均系统时间 W。这是排队论中最基本的守恒律。

**2. M/M/1系统稳定条件：**  
A. λ>μ B. λ=μ C. **λ<μ** D. 任意  
**答案：** **C**  
*解析*：M/M/1稳定要求到达率 λ 小于服务率 μ，即利用率 ρ = λ/μ < 1，否则队列无限增长。

**3. M/D/1系统中“D”表示：**  
A. 随机 B. 指数 C. **固定** D. 多服务器  
**答案：** **C**  
*解析*：Kendall记号中，D表示确定性（Deterministic）服务时间，即每个客户的服务时间为固定常数。

**4. M/M/c中c表示：**  
A. 缓冲 B. **服务器数** C. 到达率 D. 队列长度  
**答案：** **B**  
*解析*：M/M/c表示多个并行服务器（c个），每个服务率μ，到达为泊松过程。

**5. P-K公式适用于：**  
A. M/M/1 B. **M/G/1** C. M/D/1 D. M/M/c  
**答案：** **B**  
*解析*：Pollaczek-Khinchin (P-K) 公式用于计算M/G/1排队系统的平均等待时间，其中G为一般服务分布。

**6. Jackson网络核心性质：**  
A. 非线性 B. **乘积形式** C. 不稳定 D. 无解  
**答案：** **B**  
*解析*：Jackson开放网络中，各节点联合平稳分布具有乘积形式（各节点独立），从而简化分析。

**7. 排队延迟最主要来源：**  
A. 传播 B. **排队** C. 编码 D. 调制  
**答案：** **B**  
*解析*：在分组交换网络中，端到端时延主要包括排队、处理、传输、传播延迟，其中**排队延迟**是可变且主要的来源。

## 三、填空题（第1～7题）

**1. Little定理：L = [λW]**  
*解析*：L 为平均队列长度（系统中平均客户数），λ 为到达率，W 为平均系统时间。

**2. 利用率 ρ = [λ/μ]**  
*解析*：利用率定义为到达率与服务率之比，对于单服务器系统，ρ=λ/μ。

**3. M/M/1平均系统时间 W = [1/(μ-λ)]**  
*解析*：系统时间包括排队和服务时间，公式为 W = 1/(μ-λ)，其中μ>λ。

**4. 平均排队时间 Wq = [ρ/(μ-λ) = λ/(μ(μ-λ))]**  
*解析*：排队时间 Wq = W - 1/μ = 1/(μ-λ) - 1/μ = λ/(μ(μ-λ))，也等于 ρ/(μ-λ)。

**5. M/G/1中G表示[一般分布（General）]**  
*解析*：G表示任意服务时间分布，只需要已知均值和二阶矩。

**6. Burke定理输出流为[泊松过程]**  
*解析*：对于M/M/1稳定系统，输出过程是速率为λ的泊松过程，且与输入过程独立。

**7. 自相似流量的关键特征是[长程相关性（Long-Range Dependence）]**  
*解析*：自相似流量在多个时间尺度上保持突发性，其自相关函数衰减缓慢，具有长程相关特性。

## 四、多选题（第1～7题）

**1. 排队系统基本要素包括：** A.到达 B.服务 C.队列 D.调度  
**答案：** **A**、**B**、**C**、**D**  
*解析*：一个完整的排队系统由到达过程、服务过程、排队队列、调度规则（服务策略）四个基本要素组成。

**2. 网络时延组成包括：** A.传播 B.排队 C.处理 D.传输  
**答案：** **A**、**B**、**C**、**D**  
*解析*：网络节点总时延包括处理时延、排队时延、传输时延（发送时延）、传播时延，四者缺一不可。

**3. M/M/1假设包括：** A.泊松到达 B.指数服务 C.单服务器 D.固定服务  
**答案：** **A**、**B**、**C**  
*解析*：M/M/1要求：泊松到达（Markov）、指数服务（Markov）、单服务器（1），服务时间不是固定的。

**4. 影响排队性能的因素：** A.λ B.μ C.方差 D.调度策略  
**答案：** **A**、**B**、**C**、**D**  
*解析*：排队性能（如平均等待时间）受到达率λ、服务率μ、服务时间分布的方差（尤其高阶矩）以及调度策略（如FIFO, LCFS, PS）显著影响。

**5. M/G/1特点：** A.任意服务分布 B.需二阶矩 C.多服务器 D.更真实  
**答案：** **A**、**B**、**D**  
*解析*：M/G/1允许任意服务时间分布（G），P-K公式需要二阶矩，更贴近实际系统，但仍是单服务器。

**6. 自相似流量特点：** A.长相关 B.突发性 C.多尺度 D.马尔可夫  
**答案：** **A**、**B**、**C**  
*解析*：自相似流量具有长程相关性、突发性（burstiness）及多尺度（分形）特征，不是马尔可夫过程。

**7. Jackson网络适用条件：** A.泊松输入 B.独立节点 C.开放网络 D.任意分布  
**答案：** **A**、**B**、**C**  
*解析*：Jackson开放网络要求外部输入为泊松过程，各节点服务时间为指数分布，且节点之间独立，最终联合分布为乘积形式。

## 五、简答题（第1～7题）

**1. Little定理的工程意义**  
**答**：Little定理（L=λW）不依赖分布假设，提供了平均队列长度、到达率和平均延迟之间的简单关系。在工程中，可用于测量延迟（W = L/λ）或估算队列长度（L = λW），也可验证系统模型的正确性。

**2. M/M/1与M/D/1差异**  
**答**：M/M/1服务时间为指数分布（随机性强），M/D/1服务时间为常数（确定性）。相同负载下，M/D/1的平均排队时间更短，因为无随机波动；M/M/1的随机性导致更长的队列和延迟。

**3. 为什么ρ接近1会导致时延爆炸**  
**答**：ρ=λ/μ接近1时，系统接近饱和。M/M/1的平均系统时间W=1/(μ-λ)随(1-ρ)的倒数增长，分母趋近0，导致队列长度和时延趋于无穷，即时延爆炸。这是排队系统的临界现象。

**4. 多服务器模型优势**  
**答**：多服务器（M/M/c）可并行处理多个客户，在相同总服务率下（cμ），能显著降低排队延迟和队列长度，提高系统吞吐量和鲁棒性，且与单服务器相比，相同负载下性能更优。

**5. Pollaczek-Khinchin (P-K) 公式作用**  
**答**：P-K公式给出M/G/1排队系统的平均等待时间：Wq = (λ·E[S²]) / [2(1-ρ)]。它表明等待时间取决于服务时间的**二阶矩**，而不仅仅是均值，对于分析非指数服务分布的网络非常关键。

**6. Burke定理意义**  
**答**：Burke定理指出：M/M/1稳定系统的输出过程是泊松过程，且与输入独立。这使得在分析串联排队网络时，可将每个节点的输出直接作为下一节点的输入，简化了Jackson网络的分析。

**7. Jackson网络核心思想**  
**答**：Jackson开放网络由多个M/M/1（或M/M/c）节点组成，外部泊松输入，客户路由概率固定。其稳态联合分布具有**乘积形式**，即每个节点独立为M/M/1分布，从而可分别计算各节点性能。

## 六、计算题（第1～7题）

**1. M/M/1基础性能计算**  
已知：λ=6 个/秒，μ=10 个/秒  
**(1) 利用率 ρ = λ/μ = 6/10 = 0.6**  
**(2) 平均客户数 L = λ/(μ-λ) = 6/(10-6) = 6/4 = 1.5 个**  
**(3) 平均系统时间 W = 1/(μ-λ) = 1/4 = 0.25 秒**  
**(4) 平均排队时间 Wq = ρ/(μ-λ) = 0.6/4 = 0.15 秒（或 λ/(μ(μ-λ)) = 6/(10*4)=0.15）**  
*解析*：利用M/M/1标准公式，代入即可。

**2. M/M/1/3有限缓冲系统（完整推导）**  
已知：M/M/1/3，容量K=3（包括服务中），λ=4，μ=5，求阻塞概率P3。  
**答**：设状态概率P0,P1,P2,P3，平衡方程：  
λP0 = μP1 → P1 = (λ/μ)P0 = 0.8P0  
λP1 = μP2 → P2 = (λ/μ)²P0 = 0.64P0  
λP2 = μP3 → P3 = (λ/μ)³P0 = 0.512P0  
归一化：P0 + 0.8P0 + 0.64P0 + 0.512P0 = P0(1+0.8+0.64+0.512)=2.952P0=1  
→ P0 = 1/2.952 ≈ 0.3388  
阻塞概率 P_block = P3 = 0.512 * 0.3388 ≈ **0.1735**  
*解析*：有限缓冲时，丢包概率为系统满的概率。

**3. M/M/c稳定性与利用率**  
已知：M/M/3，λ=12，μ=5  
**(1) 系统利用率 ρ = λ/(c·μ) = 12/(3×5) = 12/15 = 0.8**  
**(2) 系统稳定条件：ρ < 1，此处0.8<1，故系统稳定。**  
*解析*：多服务器系统稳定性条件是总到达率小于总服务率：λ < c·μ。

**4. M/D/1排队时间计算**  
已知：M/D/1，λ=4，μ=6，服务时间恒定D=1/μ=1/6≈0.1667秒，ρ = λ/μ = 4/6 = 2/3 ≈0.6667  
**公式**：M/D/1平均排队时间 Wq = (ρ/(2μ(1-ρ)))？实际上M/D/1的P-K公式：Wq = (λ·E[S²])/(2(1-ρ))，由于E[S²] = (1/μ)² = 1/36，  
代入：Wq = (4 × (1/36)) / (2×(1-2/3)) = (4/36) / (2×(1/3)) = (1/9) / (2/3) = (1/9)×(3/2)=3/18=1/6≈0.1667秒  
或者更简单：M/D/1的 Wq = ρ/(2μ(1-ρ)) = (2/3)/(2×6×(1/3)) = (2/3)/(4)= 2/12=1/6≈0.1667  
**答**：Wq = 1/6 ≈ 0.1667 秒  
*解析*：利用P-K公式，二阶矩为(1/μ)²。

**5. M/G/1 (P-K公式完整计算)**  
已知：λ=3，E[S]=0.2，Var(S)=0.04，则E[S²] = Var + (E[S])² = 0.04 + 0.04 = 0.08  
ρ = λ·E[S] = 3×0.2 = 0.6  
**公式**：Wq = (λ·E[S²]) / [2(1-ρ)] = (3×0.08) / (2×(1-0.6)) = 0.24 / (2×0.4) = 0.24 / 0.8 = 0.3 秒  
**答**：平均排队时间 Wq = 0.3 秒  
*解析*：P-K公式适用于任意服务分布，需已知一阶矩和二阶矩。

**6. Jackson网络（端到端时延）**  
两节点串联：外部输入λ=5，节点1：μ1=8，节点2：μ2=10，均为M/M/1。  
节点1负载 ρ1 = λ/μ1 = 5/8 = 0.625，节点2负载 ρ2 = λ/μ2 = 5/10 = 0.5（因无分流，节点2到达率仍为5）  
节点1平均时间 W1 = 1/(μ1-λ) = 1/(8-5)=1/3≈0.3333秒  
节点2平均时间 W2 = 1/(10-5)=1/5=0.2秒  
**系统平均总时延 W = W1 + W2 = 0.3333 + 0.2 = 0.5333秒**  
*解析*：Jackson开放网络各节点独立计算，端到端延迟为各节点时延之和。

**7. 系统设计优化（约束反推）**  
M/M/1，μ=10，要求 W ≤ 0.2 秒，求最大 λ。  
由 W = 1/(μ-λ) ≤ 0.2 → μ-λ ≥ 1/0.2 = 5 → 10-λ ≥ 5 → λ ≤ 5  
同时需满足 λ < μ=10，故最大允许到达率 λ_max = 5 个/秒。  
*解析*：根据平均系统时间公式反解，得到λ上限。
